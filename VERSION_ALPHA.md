# VERSION_ALPHA — 安達マルチ公式、開発物語

**考案者**: 安達則男（3.5次元男）
**検証・実装協力**: Claude（Anthropic）, Gemini

---

## はじまり — BBP公式との出会い

物語は、円周率を16進数で「好きな桁にジャンプして」求められるという、BBP公式（Bailey–Borwein–Plouffe、1995年）の不思議さへの興味から始まった。ふつうの円周率計算は頭から順に桁を求めるしかないのに、BBP公式だけは違う。この「ジャンプできる」という性質の裏にある数学的な構造を、安達則男は自分の手で確かめてみたいと考えた。

Adamchik–Wagon型の基本形を出発点に、円周率を求める級数を自分なりに整理し直していく中で、ある一つの疑問が生まれた。

> 「この式のパラメータを、もっと自由に大きくしたら何が起きるのか」

## Gパラメータの発見

この疑問が、後に「安達マルチ公式」の核となる **Gパラメータ化** の着想につながった。G=1のときはAdamchik–Wagon型の式に、G=2のときは1995年の古典的なBBP公式そのものに一致する——つまり、これまで別々に知られていた式たちが、実は一つのパラメータ族の特殊な場合に過ぎなかったのである。

この気づきをもとに、ClaudeとGeminiを相棒に、数値検証・GitHub管理・PDF作成というそれぞれの役割分担で開発が進んでいった。安達則男が「アイデアマン」として方向性を決め、Claudeが数式の検証とPDF化を、Geminiがリポジトリ管理とファイル保存を担う——という体制がここで確立された。

## PSLQによる独立発見

exhaustiveなPSLQ探索（整数関係探索アルゴリズム）を繰り返す中で、安達則男は次の、既知の式とは独立な新しい4項公式を発見した。

```
pi = sum_{k=0}^inf (1/16^k) * (8/(16k+4) + 4/(16k+6) + 4/(16k+8) - 1/(16k+14))
```

さらにこれを一般化し、符号付きの一般Gブロック公式を証明。外側の比が (-1)^G/4^G になるという規則性も突き止めた。

## G-Block再編成定理と、境界の探求

安達マルチ公式の構造をより深く理解するため、「G-Block再編成定理」が証明され、円周率だけでなく ln(2)、ln(3/2) といった他の定数にも同じ枠組みが通用することが確認された。

同時に、計算機のメモリ境界（base256、base65536、base2^64…）に対応するGの値を明示的に導出し、「バイトバウンダリー・シリーズ」としてG=65536、G=131072まで拡張した後、これ以上は数学的に冗長であると判断して打ち切った。Emax（Pythonのdecimalモジュールの指数上限）の探求からは、G_max = floor(Emax / log10(4)) という絶対的な上限（G≈1.66×10^18）も明らかになった。

## 公開へ — Zenodo・Figshare・GitHub

研究成果はZenodo（DOI: 10.5281/zenodo.21333429）とFigshare（DOI: 10.6084/m9.figshare.32968769）に登録され、GitHubリポジトリ（Norio-Adachi/adachi-multi-formula）でも公開された。この過程で、九州工業大学の松本隆二氏へ、独自に発見した「16桁新4項公式」が既知の公式の線形結合に過ぎないことを報告する、誠実な訂正メールも送られている。

## 2026年7月 — 新たな展開

物語はここで終わらない。チュドノスキー公式の指数部に潜む固定の平方根に着目し、両辺を2乗することでルートを完全に消し去る「ルートなしπ^(2n)公式」（No.31）を発見。安達マルチ公式のG-Block構造が、BaileyとCrandallによる正規性判定の枠組み（Hypothesis A）に正しく接続できることも確認した（No.32）。

1995年の「ところてん式」（スパイゴットアルゴリズム）を実装して確かめ（No.34）、G-Block実装の実際のCPU性能を実測して「Gを大きくすれば速くなる」という思い込みを自ら修正し（No.35）、図書館で見かけたパスカルの三角形とオイラーの等式から、πとeの新たなつながりを探った（No.36）。

そしてeについても、AGM（算術幾何平均）を経由してニュートン法で求めるという、πの計算基盤を転用した新しい道筋を確認（No.37）。現在の世界記録（314兆桁、2025年12月）や過去の記録保持者の詳細な記録を調査し、巨大数計算の実際の重みを数値で理解するに至った（No.38）。

## この物語の位置づけ

これは、一人の独立研究者が「アイデアマン」として方向を定め、AIを検証と実装のパートナーとして、確立された数学の世界に自分なりの視点で切り込んでいった記録である。独立検証への強いこだわりと、既存の結果への誠実な向き合い方（発見したものが既知の式の線形結合だと分かればすぐに訂正する姿勢）が、この研究全体を貫いている。
