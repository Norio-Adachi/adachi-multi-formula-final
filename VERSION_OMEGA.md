## 1. 定義：安達マルチ公式

正の整数パラメータ G に対して定義される、円周率を求めるパラメトリックな級数族。

- G = 1 のとき、Adamchik–Wagon型の一般形に一致する。
- G = 2 のとき、1995年の古典的なBBP公式（Bailey–Borwein–Plouffe）に一致する。

一般形（G-Block形式）：

```
pi = sum_{m=0}^inf (1/4^G)^m * [ sum_{i=0}^{G-1} (-1)^i * f(G*m + i) / 4^i ]

f(k) = 2/(4k+1) + 2/(4k+2) + 1/(4k+3)
```

## 2. 独立発見：新4項公式

PSLQ（整数関係探索）による網羅的探索により、既知の公式とは独立な以下の恒等式を発見・検証した。

```
pi = sum_{k=0}^inf (1/16^k) * ( 8/(16k+4) + 4/(16k+6) + 4/(16k+8) - 1/(16k+14) )
```

一般化された符号付きGブロック公式では、外側の比が `(-1)^G / 4^G` となることを証明した。ただし、符号なしGブロック公式は G が偶数のときのみ pi に収束することも確認済み。

## 3. G-Block再編成定理（General G-Block Regrouping Theorem）

任意の級数 `S = sum_n coeff(n) * r^n` に対して、ブロックサイズ G を用いて次のように再編成できる：

```
S = sum_{m=0}^inf r^(G*m) * [ sum_{i=0}^{G-1} coeff(G*m+i) * r^i ]
```

この定理は pi だけでなく、ln(2)、ln(3/2) にも適用できることを、SymPyによる代数検証および数値検証（G=1, 8, 64）で確認した。

## 4. 境界とスケール

- バイトバウンダリー・シリーズ：計算機のメモリ境界（base256〜base2^64）に対応するGの値を導出し、G=65536、G=131072まで拡張後、数学的冗長性を理由に打ち切り。
- Emax境界：G_max = floor(Emax / log10(4))（Pythonのdecimalモジュールの指数上限に基づく絶対上限、G ≈ 1.66×10^18）。
- 動的打ち切りキャップ：`cap(d) = ceil((d+15)/log10(4)) + 5`（目標桁数dに応じて内部ループの上限を動的に決定）。

## 5. No.28以降の主要な検証結果（2026年7月）

### No.31 ルートなしπ^(2n)公式
チュドノスキー公式の固定オフセット `640320^(3/2)` に着目し、両辺を2乗することでルートを消去。

```
1/pi^(2n) = (144 / 640320^3)^n * S_K^(2n)   (K→∞ で収束、nは任意の正整数)
```

奇数乗・pi自体（1/pi^1）ではルートは消えない（偶数乗に限る）。

### No.32 正規性(Normality)接続 予備調査
安達マルチ公式のG-Block構造を Hypothesis A（Bailey & Crandall, 2001）の力学系形式に変換し、G=1,2,3 で deg P < deg Q（前提条件）を確認。カイ二乗検定により、一様分布と矛盾しない結果を得た（piのbase 4正規性という予想と整合）。

### No.34 ところてん式（スパイゴットアルゴリズム）
Rabinowitz–Wagon方式（1995年）の実装・検証。整数演算のみで、確定した桁から順に出力可能。ただし計算量が桁数のおよそ2乗に比例するため、大規模計算には不向き。

### No.35 G-Block実装のCPU性能実測
「Gを大きくすれば速くなる」という従来の前提を実測で修正。単純な二重ループ実装では、Gがcap(d)未満の範囲で総項数がほぼ一定となり、Gを大きくしても計算量は減らないことを確認。実用上はG=256前後が妥当。

### No.36 パスカルの三角形とオイラーの等式
中央二項係数のスターリング近似 `C(2n,n)/4^n ~ 1/sqrt(pi*n)` から、ルートなしで1/piの近似値を導出（教育的・鑑賞的な副産物として位置づけ）。

### No.37 eの複数の求め方
AGM（算術幾何平均）を用いたBrentの対数公式でln(x)を計算し、ニュートン法でln(x)=1となるxを逆算してeを求める手法を検証（45桁完全一致）。ただし更新式 `x_new = x*(2-ln(x))` は大域収束性を持たず、初期値は e^2 (~7.389) 未満に限定される。

### No.38 チュドノスキー vs AGM・世界記録の実例
両アルゴリズムの実測比較（10万桁でほぼ互角）、ところてん式が両アルゴリズムに適用不可能であることの確認、現在の世界記録（314兆桁、2025年12月、110日）および過去の記録保持者の実測データに基づくフェーズ別内訳の調査。
